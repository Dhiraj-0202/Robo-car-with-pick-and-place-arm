Introduction

• Mobile manipulators combine a mobile robot base and a robotic arm to perform tasks like picking, placing, and transporting objects.


• These systems are widely used in industrial automation, warehouses, and service robotics.


• The main aim of this project is to design and develop a mobile manipulator capable of performing pick-and-place operations.


• A simulation model was developed using Python and Pygame to test inverse kinematics and arm movement before implementing the hardware.


• This project helps understand robot kinematics, motion control, and automation concepts through both simulation and practical implementation.

Objectives

• To design and develop a mobile manipulator system capable of performing pick-and-place operations.


• To implement a simulation model using Python and Pygame to visualize the robotic arm movement and test inverse kinematics.


• To apply inverse kinematics and motion control techniques for accurate positioning of the robotic arm.


• To demonstrate the coordination between the mobile platform and the robotic arm for object handling.

Hardware Components Used

1. Microcontroller (ESP32)

The ESP32 acts as the main controller of the robot. It processes commands received from the mobile application via Bluetooth and controls the motors, servo motors, ultrasonic sensor, and gripper mechanism.

2. Ultrasonic Sensor (HC-SR04)

The HC-SR04 ultrasonic sensor detects nearby objects by measuring distance using ultrasonic sound waves. It helps the robot determine when an object is within the picking range.

3. DC Motors

DC motors drive the mobile robot base and allow movement in different directions such as forward, backward, left, and right.

4. Motor Driver Module (L298N)

The L298N motor driver controls the direction and speed of the DC motors and acts as an interface between the ESP32 controller and the motors.

5. Servo Motors (MG996R)

High torque servo motors such as MG996R are used to control the joints of the robotic arm and the gripper mechanism, providing precise angular movement.

6. Robotic Arm Links

The robotic arm consists of two mechanical links forming a two-link manipulator structure that allows the arm to reach the object.

7. Gripper Mechanism

The gripper is the end-effector attached to the robotic arm. It opens and closes using a servo motor to pick and place objects.

8. Robot Chassis with Wheels

A four-wheel robotic chassis forms the mobile base of the system and supports the robotic arm, motors, battery, and electronic components.

9. Battery / Power Supply

A rechargeable battery pack supplies power to the ESP32 controller, motors, and servo motors to operate the entire robotic system.

System Architecture

• Mobile Base Unit – Provides movement for the robot in the workspace using motors and wheels.

• Robotic Arm Mechanism – A two-link robotic arm mounted on the mobile base for reaching and manipulating objects.

• Gripper Mechanism – End-effector used to pick and place objects securely.

• Control Unit – Microcontroller or controller used to process commands and control motors and servos.

• Simulation Module – Python and Pygame environment used to model the robot, test inverse kinematics, and visualize motion before hardware implementation.

• Power Supply System – Provides required power to motors, servos, and control electronics.

• User Interface / Input Control – Allows the user to control the mobile base and robotic arm movements.

Hardware Implementation

Start

The system is powered ON and the robot begins initialization.

Step 1 – Power Supply

The battery supplies power to the ESP32 controller, servo motors, DC motors through the L298N motor driver module, and the HC-SR04 ultrasonic sensor.

Step 2 – System Initialization

The ESP32 initializes all connected components including the motor driver, ultrasonic sensor, and servo motors. Bluetooth communication is then activated to receive commands from the mobile application.

Step 3 – Mobile App Control

The user connects a mobile application to the ESP32 via Bluetooth and sends movement commands to control the robot.

Step 4 – Robot Navigation

The robot base moves according to commands received from the mobile application while the ultrasonic sensor continuously monitors nearby objects.

Step 5 – Object Detection

When the ultrasonic sensor detects an object within a predefined distance, the ESP32 stops the robot near the object.

Step 6 – Arm Movement

The robotic arm moves to the predefined picking position using servo motors controlled by the ESP32.

Step 7 – Object Picking

The gripper closes to securely grasp the detected object.

Step 8 – Object Transportation

The robot is navigated to the destination location using the mobile application controls.

Step 9 – Object Placement

The robotic arm moves to the placing position and the gripper opens to release the object.

End

The pick-and-place operation is completed successfully.

Simulation Implementation

Step 1: Simulation Environment

  • A simulation environment is developed using Python and Pygame.
  • The simulation visually represents the robot base, arm links, joints, gripper, objects, and workspace grid.
  • The robotic arm is modeled as a two-link planar manipulator attached to the mobile base.
  • Objects are placed at different (x, y) coordinates in the workspace.

Step 2: Robot Navigation

  • The mobile robot base is controlled using the keyboard arrow keys.
  • Using these keys, the robot can move forward, backward, left, and right in the workspace to approach the object.
  • When the robot moves near the object, the object position (x, y) relative to the robotic arm is determined.
  • This position is used as the target coordinate for the robotic arm.

Step 3: Inverse Kinematics

Inverse kinematics is used to calculate the required joint angles so that the robotic arm can reach the target position (x, y).

  • These equations determine the joint angles of the robotic arm.
  • The calculated angles control the servo motors to reach the object.

Step 4: Robotic Arm Movement

  • The robotic arm moves according to the calculated joint angles.
  • When the end-effector reaches the object location, the gripper closes to pick the object.

Step 5: Object Placement

  • After picking the object, the robotic arm moves to the destination position.
  • The robot movement is controlled using the keyboard arrow keys.
  • By pressing the P key, the gripper opens to place the object at the destination location.

Challenges

• Inverse kinematics implementation – Calculating accurate joint angles for the robotic arm to reach a specific target position required correct mathematical modeling and careful implementation.


• Smooth arm movement – Achieving smooth and realistic arm motion required gradual angle updates to simulate the behavior of servo motors.


• Object detection and pickup logic – Ensuring that the end-effector correctly detects when it reaches the object and triggers the gripper mechanism.


• Synchronization between arm and mobile base – Coordinating the movement of the mobile robot platform and the robotic arm within the workspace.


• Simulation accuracy – Designing the simulation so that it closely represents the behavior of the real hardware system while maintaining smooth performance.

Results

• A simulation model of the mobile manipulator system was successfully developed using Python and Pygame.

• The inverse kinematics algorithm correctly calculated the joint angles required for the robotic arm to reach the target coordinates.

• The robotic arm demonstrated smooth and continuous motion, simulating realistic servo motor movement.

• The end-effector successfully reached the objects placed in the workspace.

• The gripper mechanism effectively performed pick-and-place operations within the simulation environment.

• The simulation displayed important parameters such as joint angles (θ₁, θ₂) and end-effector position.

• The workspace visualization helped in understanding the reachable area of the robotic arm.

• The simulation results confirmed that the control logic and inverse kinematics implementation work correctly.
